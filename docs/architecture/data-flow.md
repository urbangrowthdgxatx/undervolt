## Overview

Undervolt processes 2.4M+ construction permits from Austin Open Data through a multi-stage pipeline, then serves insights via a Next.js frontend with local LLM integration.

```
Austin Open Data (1.5GB CSV)
    ↓
Python Pipeline (Feature Extraction + Clustering)
    ↓
Processed CSVs (3.7GB)
    ↓
Next.js API Routes (Caching + LLM)
    ↓
Frontend UI (Story Builder + Map & Chat)
```

## Data Sources

### 1. Raw Source Data

**Location**: `data/Issued_Construction_Permits_20251212.csv`

**Source**: [Austin Open Data Portal](https://data.austintexas.gov/Building-and-Development/Issued-Construction-Permits/3syk-w9eu)

**Stats**:
- **Size**: 1.5GB
- **Records**: 2,458,645 permits
- **Columns**: 66 fields
- **Coverage**: Construction permits issued 2015-2025

**Key Fields**:
```
permit_num              Unique identifier (e.g., "2025-160307 PP")
permit_type             Type code (BP, EP, MP, PP, DP)
permit_type_desc        Description (Building, Electrical, Mechanical, Plumbing, Demolition)
description             Free-text permit description
issued_date             Date permit was issued
calendar_year_issued    Year (2015-2025)
latitude, longitude     Geocoded location
original_zip            ZIP code
council_district        Austin council district (1-10)
work_class              Work classification (New, Remodel, Repair, etc.)
total_job_valuation     Dollar value of work
housing_units           Number of units affected
```

**Download**:
```bash
bash scripts/download_data.sh
# OR
python scripts/download_data.py
```

## Processing Pipeline

### Pipeline Architecture

```
┌────────────────────────────────────────────────────────────┐
│                  EXTRACTION PIPELINE                       │
│                                                            │
│  ┌─────────────┐                                           │
│  │   CSV       │  2.4M permits                             │
│  │  (1.5GB)    │  Issued_Construction_Permits.csv          │
│  └──────┬──────┘                                           │
│         │                                                  │
│         ▼                                                  │
│  ┌─────────────┐                                           │
│  │  Stage 1    │  Load + Clean (pandas/cuDF)               │
│  │  Clean      │  • Select columns                         │
│  │             │  • Drop nulls, min length filter          │
│  └──────┬──────┘  → 2.4M rows                              │
│         │                                                  │
│         ▼                                                  │
│  ┌─────────────┐                                           │
│  │  Stage 2    │  NLP Keyword Extraction                   │
│  │  Extract    │  • Regex matching on description          │
│  │             │  • 80 boolean features                    │
│  │             │  • Keywords: solar, battery, ev, etc.     │
│  └──────┬──────┘  → Feature-enriched data                  │
│         │                                                  │
│         ▼                                                  │
│  ┌─────────────┐                                           │
│  │  Stage 3    │  ML Clustering (scikit-learn)             │
│  │  Cluster    │  • TF-IDF vectorization                   │
│  │             │  • K-Means clustering                     │
│  │             │  • Cluster naming                         │
│  └──────┬──────┘  → Clustered permits                      │
│         │                                                  │
│         ▼                                                  │
│  ┌─────────────┐                                           │
│  │  Stage 4    │  Write Output                             │
│  │  Save       │  • CSV files (enriched, summary)          │
│  │             │  • JSON metadata (cluster names)          │
│  └─────────────┘  → output/ directory                      │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### Running the Pipeline

```bash
# Option 1: New modular structure
python run_pipeline.py

# Option 2: Legacy monolithic script
python pipeline_cudf.py
```

**Processing Time**:
- Stage 1 (Load/Clean): ~30 seconds
- Stage 2 (NLP Extract): ~60 seconds
- Stage 3 (Clustering): ~90 seconds
- Stage 4 (Save): ~30 seconds
- **Total**: ~3-4 minutes for 2.4M permits

#### 2. Story Builder (`/story`)

**File**: `frontend/src/app/story/page.tsx`

**Data Sources**:

1. **Hardcoded Storylines** → `frontend/src/components/StorylineCards.tsx`
   - 3 storylines: Resilience, Solar, Remodel
   - Each has 28-36 pre-written questions

2. **LLM-Generated Questions** → `/api/story/suggest`
   - Called after 2+ cards added
   - Generates 20-28 new questions based on story context
   - Response time: ~5 seconds

3. **Chat Responses** → `/api/chat`
   - LLM-powered insights
   - Response time: 15-41 seconds (streaming)
   - Currently uses fallback keyword matching

**User Flow**:
```
Page Load
  ↓
Onboarding Wizard (localStorage check)
  ↓
Storyline Selection (3 cards)
  ↓
Floating Questions (28-36 initial questions from storyline)
  ↓
Click Question → Call /api/chat
  ↓
Add Card to Story
  ↓
After 2+ cards → Call /api/story/suggest (new questions)
```

**File**: `frontend/src/app/dashboard/page.tsx`

**Data Sources**:

1. **Stats API** → `/api/stats`
   - Reads from `.stats-cache.json`
   - Fallback: streams `permit_summary_by_zip.csv`
   - Response time: 58ms (cached), ~60s (uncached)

2. **Chat API** → `/api/chat`
   - Same as Story Builder

3. **Map Data** → *(planned)*
   - GeoJSON from permit locations

**Performance**:

## API Endpoints

### 1. `/api/stats` (GET)

**Purpose**: Return aggregated permit statistics

**File**: `frontend/src/app/api/stats/route.ts`

**Data Flow**:
```
Request
  ↓
Check memory cache (globalForStats.statsCache)
  ↓ (miss)
Check disk cache (.stats-cache.json)
  ↓ (miss)
Stream CSV (permit_summary_by_zip.csv)
  ↓
Process 2.4M rows:
  • Count clusters
  • Count ZIPs
  • Count energy features (solar, hvac, electrical, etc.)
  ↓
Cache to memory + disk
  ↓
Return JSON
```

**Response**:
```json
{
  "totalPermits": 2405158,
  "clusterDistribution": [...],
  "topZips": [...],
  "energyStats": {
    "solar": 0,
    "hvac": 42211,
    "electrical": 82657,
    "newConstruction": 130359,
    "remodels": 44081
  },
  "clusterNames": {...},
  "lastUpdated": "2025-12-30T17:01:40.435Z"
}
```

**Performance**:
- Memory cache hit: 11-16ms
- Disk cache hit: 58ms
- CSV processing: ~60 seconds

**Clear Cache**:
```bash
curl -X POST http://localhost:3000/api/stats
```

### 2. `/api/story/suggest` (POST)

**Purpose**: Generate contextual questions using local LLM

**File**: `frontend/src/app/api/story/suggest/route.ts`

**Request**:
```json
{
  "blocks": [],
  "mode": "scout"
}
```

**Data Flow**:
```
Request with existing story blocks
  ↓
Build context from block headlines
  ↓
Call Ollama API (http://localhost:11434)
  • Model: llama3.2:3b
  • Temperature: 0.8
  • num_predict: 400 tokens
  • keep_alive: 30m (GPU-cached)
  ↓
Parse LLM response
  • Split by newlines
  • Remove numbering
  • Take first 28 questions
  ↓
Return JSON
```

**Response**:
```json
{
  "questions": [
    "Where is solar growing?",
    "Which areas have batteries?",
    "Solar-to-storage ratio?",
    ...
  ]
}
```

**Performance**: 4.7-5.0 seconds

**Fallback**: Returns hardcoded DEFAULT_QUESTIONS on error

### 3. `/api/chat` (POST)

**Purpose**: Answer user questions with insights

**File**: `frontend/src/app/api/chat/route.ts`

**Request**:
```json
{
  "message": "Where are generators?",
  "mode": "scout",
  "existingBlocks": []
}
```

**Data Flow**:
```
Request with question
  ↓
Call local LLM (Ollama)
  ↓
Stream response (Server-Sent Events)
  ↓
Parse into StoryBlock format
  ↓
Return structured JSON
```

**Response** (streaming):
```
event: block
data: {"headline":"...","insight":"...","confidence":"high"}

event: done
data: {}
```

**Performance**: 15-41 seconds (depends on LLM response length)

**Current State**: Uses fallback keyword matching (hardcoded responses)

### 4. `/api/chat-llm` (POST)

**Purpose**: Direct LLM chat (GPU-accelerated)

**File**: `frontend/src/app/api/chat-llm/route.ts`

**Performance**: ~1.7s with GPU cache

## Known Issues

### 1. Incorrect ZIP Code Data

**Problem**: Stats cache contains invalid ZIP codes

**Evidence**:
```json
"topZips": [
  { "zip": "No", "count": 162691 },    // Invalid
  { "zip": "0", "count": 81897 },       // Invalid
  { "zip": "Yes", "count": 17566 },     // Invalid
  { "zip": "78746", "count": 3388 }     // Valid
]
```

**Root Cause**: CSV parsing doesn't validate ZIP format

**Location**: `scripts/precompute_stats.js` line 72-75

**Fix**:
```javascript
// Current (wrong):
const zip = row.zip_code;
if (zip) {
  zipCounts[zip] = (zipCounts[zip] || 0) + 1;
}

// Should be:
const zip = row.zip_code;
if (zip && /^\d{5}$/.test(zip)) {  // Validate 5-digit ZIP
  zipCounts[zip] = (zipCounts[zip] || 0) + 1;
}
```

### 2. LLM Not Data-Grounded

**Problem**: LLM generates questions from general knowledge, not actual permit data

**Impact**: Questions may reference patterns that don't exist in the dataset

**Future**: Implement "Context Bundles" (see README.md lines 347-376)

**Solution**: Pre-compute data-grounded questions after pipeline extraction:
```
Extract signals → Analyze patterns → Generate questions from actual data
```

### 3. Cluster Distribution Shows Single Cluster

**Problem**: All permits assigned to cluster 0

**Evidence**:
```json
"clusterDistribution": [
  {
    "id": 0,
    "name": "New Residential Construction",
    "count": 2405158,
    "percentage": 100
  }
]
```

**Root Cause**: Clustering pipeline may not have run, or cluster column not populated

**Fix**: Re-run pipeline with clustering enabled
